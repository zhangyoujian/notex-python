# Notex API 接口手册

> 说明：本文档基于仓库当前实现整理 `routers/*` 对外 HTTP 接口，并结合前端调用方式补齐字段类型与入参形式。

## 1. 基础信息

### 1.1 服务地址

- 默认监听：`http://<SERVER_HOST>:<SERVER_PORT>`  
- 代码默认值（可被环境变量覆盖）：`SERVER_HOST=0.0.0.0`、`SERVER_PORT=8080`

### 1.2 统一响应体（成功）

所有成功接口通过 `success_response` 返回统一结构：

```json
{
  "code": 200,
  "message": "success",
  "data": { }
}
```

- `code`：业务码（多个接口会在响应体里填写 `201/204` 等）
- `message`：成功消息
- `data`：业务数据（可能为 `null`）

### 1.3 统一响应体（错误）

全局异常处理器返回同样结构：

#### 1) `HTTPException`

```json
{
  "code": 401,
  "message": "无效的令牌或已经过期的令牌",
  "data": null
}
```

#### 2) `IntegrityError`（数据库约束）

```json
{
  "code": 400,
  "message": "用户名已存在",
  "data": {
    "error_type": "IntegrityError",
    "error_detail": "...",
    "path": "http://.../..."
  }
}
```

#### 3) 其他未捕获异常 / 数据库异常

```json
{
  "code": 500,
  "message": "服务器内部错误",
  "data": {
    "error_type": "...",
    "error_detail": "...",
    "traceback": "...",
    "path": "http://.../..."
  }
}
```

> 前端在 `fetch` 失败时主要读取 `message` 字段。

### 1.4 鉴权方式

需要鉴权的接口，使用请求头：

- `Authorization: Bearer <token>`

token 获取方式：
- 调用 `POST /api/auth/login` 或 `POST /api/auth/register`
- 响应体 `data.token` 为 token 值

token 解析方式：
- 后端会把 header 值中的 `Bearer ` 前缀去掉后使用（即 header 形如 `Bearer abc...`）

管理员权限：
- 管理员接口额外校验当前用户邮箱是否等于 `admin@zuel.edu.cn`

## 2. 接口明细（HTTP API）

### 2.1 健康检查与配置

#### GET `/api/health`

- 鉴权：需要（路由依赖 `get_current_user`）
- 入参：
  - 无
- 出参（成功，`data`）：
  - `status`：string（固定 `ok`）
  - `version`：string（固定 `1.0.0`）
  - `timestamp`：number（`time.monotonic()`）
  - `services`：object
    - `vector_store`：string（如 `chroma`）
    - `llm`：string（当前 openai 模型名）

#### GET `/api/config`

- 鉴权：需要
- 入参：
  - 无
- 出参（成功，`data`）：
  - `{}`（空对象）

### 2.2 认证接口

#### GET `/api/auth/me`

- 鉴权：需要
- 入参：
  - Header：`Authorization: Bearer <token>`
- 出参（成功，`data`）：
  - `id`：number（用户 id）
  - `email`：string
  - `username`：string
  - `avatar`：string（头像 URL）

可能错误：
- 401：token 无效/过期（`message` 为“无效的令牌或已经过期的令牌”）

#### POST `/api/auth/register`

- 鉴权：不需要
- 入参（JSON body，`RegisterRequest`）：
  - `email`：string（邮箱）
  - `username`：string（用户名）
  - `password`：string（密码，需满足复杂度校验；不通过返回 400）
- 出参（成功，`data`）：
  - `token`：string（认证令牌）
  - `userInfo`：object
    - `username`：string
    - `email`：string
    - `avatar`：string | null

可能错误（HTTP 400）：
- 用户已存在
- 邮箱格式无效
- 密码复杂度不满足（返回复杂度校验 detail）

#### POST `/api/auth/login`

- 鉴权：不需要
- 入参（JSON body，`LoginRequest`）：
  - `email`：string
  - `password`：string
- 出参（成功，`data`）：
  - `token`：string
  - `userInfo`：object（同 `register`）

可能错误：
- 401：用户名或密码错误

### 2.3 文件上传

#### POST `/api/upload`

- 鉴权：需要
- 入参（`multipart/form-data`）：
  - Header：`Authorization: Bearer <token>`
  - `file`：binary（上传文件；后端使用字段名 `file`）
  - `notebook_id`：string（所属笔记本 id）
- 出参（成功，`data`，`Source` 信息）：
  - 见下文 `GET /api/notebooks/{notebook_id}/sources` 的 `data[]` 字段结构

可能错误：
- 403：笔记本无权限
- 500：保存上传目录失败 / 提取文件失败（取决于实现细节）

### 2.4 笔记本（Notebooks）

> 以下接口均需要鉴权（`Authorization: Bearer <token>`）。

#### GET `/api/notebooks`

- 入参：
  - 无（由 token 决定用户）
- 出参（成功，`data`，数组）：
  - `data[]`：NotebookInfo（字段见下）

NotebookInfo 字段：
| 字段 | 类型 | 描述 |
|---|---|---|
| `id` | string | 笔记本 id |
| `user_id` | number | 所属用户 id |
| `name` | string | 笔记本名称 |
| `description` | string | 笔记本描述 |
| `is_public` | number | 是否公开（0/1） |
| `public_token` | string | 公开访问令牌 |
| `created_at` | string | 创建时间（ISO 8601） |
| `updated_at` | string | 更新时间（ISO 8601） |
| `metadata` | object | 元数据（通常为 JSON 对象） |

#### GET `/api/notebooks/stats`

- 出参（成功，`data`，数组）：
  - 与 `GET /api/notebooks` 类似，额外包含：
    - `source_count`：number（来源数量）
    - `note_count`：number（笔记数量）

#### POST `/api/notebooks`

- 入参（JSON body，`NotebookRequest`）：
  - `name`：string（必填）
  - `description`：string | null（选填）
- 出参（成功，`data`）：
  - NotebookInfo（同上）

#### GET `/api/notebooks/{notebook_id}`

- Path 参数：
  - `notebook_id`：string
- 出参（成功，`data`）：
  - NotebookInfo

可能错误：
- 404：Notebook not found
- 403：Access denied（非本人笔记本）

#### PUT `/api/notebooks/{notebook_id}`

- Path 参数：
  - `notebook_id`：string
- 入参（JSON body，`NotebookRequest`）：
  - `name`：string
  - `description`：string | null
- 出参（成功，`data`）：
  - NotebookInfo（更新后的）

#### DELETE `/api/notebooks/{notebook_id}`

- Path 参数：
  - `notebook_id`：string
- 出参（成功）：
  - `data`：null

可能错误：
- 404 / 403：同权限校验

#### PUT `/api/notebooks/{notebook_id}/public`

- Path 参数：
  - `notebook_id`：string
- 入参（JSON body，注意 embed=True）：
  - `is_public`：boolean（设置为公开/私有）
  - 由于后端 `Body(..., embed=True)`，请求体必须为：`{"is_public": true}`
- 出参（成功，`data`）：
  - NotebookInfo（更新后的）

### 2.5 来源（Sources）

#### GET `/api/notebooks/{notebook_id}/sources`

- Path 参数：
  - `notebook_id`：string
- 出参（成功，`data`，数组）：
  - `data[]`：SourceInfo

SourceInfo 字段：
| 字段 | 类型 | 描述 |
|---|---|---|
| `id` | string | 来源 id |
| `notebook_id` | string | 所属笔记本 id |
| `name` | string | 来源名称 |
| `type` | string | 来源类型（如 `text/url/file/...`） |
| `url` | string | URL（若为 URL 来源，否则为空） |
| `content` | string | 内容（若为文本/抓取后的内容，否则为空） |
| `file_name` | string | 文件名（若为文件来源） |
| `file_size` | number | 文件大小（若为文件来源） |
| `chunk_count` | number | 分块数量 |
| `created_at` | string | 创建时间 |
| `updated_at` | string | 更新时间 |
| `metadata` | object | 元数据（JSON 对象） |

可能错误：
- 403：非本人笔记本（Access denied）

#### POST `/api/notebooks/{notebook_id}/sources`

- Path 参数：
  - `notebook_id`：string
- 入参（JSON body，`SourceRequest`）：
  - `name`：string（必填）
  - `type`：string（必填）
  - `url`：string | null（选填；当提供时后端会尝试抓取并写入 `content`）
  - `content`：string | null（选填；当 type 为文本/或抓取完成的内容）
  - `metadata`：object | null（选填；默认 `{}`）
- 出参（成功，`data`）：
  - SourceInfo

可能错误：
- 500：无效 URL 或被禁止抓取（detail 为“无效的URL或者当前URL被禁止爬取”）

#### DELETE `/api/notebooks/{notebook_id}/sources/{source_id}`

- Path 参数：
  - `notebook_id`：string
  - `source_id`：string
- 出参（成功）：
  - `data`：null

可能错误：
- 404：Source not found

### 2.6 笔记（Notes）

#### GET `/api/notebooks/{notebook_id}/notes`

- Path 参数：
  - `notebook_id`：string
- 出参（成功，`data`，数组）：
  - `data[]`：NoteInfo

NoteInfo 字段：
| 字段 | 类型 | 描述 |
|---|---|---|
| `id` | string | 笔记 id |
| `notebook_id` | string | 所属笔记本 id |
| `title` | string | 笔记标题 |
| `content` | string | 笔记内容（infograph 可能为空） |
| `type` | string | 笔记类型（summary/faq/study_guide/...） |
| `source_ids` | string | 关联来源 id 列表（数据库中为 JSON 数组字符串） |
| `created_at` | string | 创建时间 |
| `updated_at` | string | 更新时间 |
| `metadata_` | object | 元数据（注意字段名带下划线：`metadata_`） |

#### POST `/api/notebooks/{notebook_id}/notes`

- Path 参数：
  - `notebook_id`：string
- 入参（JSON body，`NoteRequest`）：
  - `title`：string
  - `content`：string
  - `type`：string
  - `source_ids`：string  
    - 期望为“来源 id 列表的 JSON 字符串”，例如：`["src1","src2"]`
- 出参（成功，`data`）：
  - NoteInfo

> 注：前端当前主要通过 `POST /api/notebooks/{notebook_id}/transform` 生成笔记，而不是直接调用该创建接口。

#### DELETE `/api/notebooks/{notebook_id}/notes/{note_id}`

- Path 参数：
  - `notebook_id`：string
  - `note_id`：string
- 出参（成功）：
  - `data`：null

可能错误：
- 403：Access denied

### 2.7 转换（Transform）

#### POST `/api/notebooks/{notebook_id}/transform`

- Path 参数：
  - `notebook_id`：string
- 入参（JSON body，`TransformationRequest`）：
  - `type`：string（必填；支持见项目转换类型映射）
    - 可用类型（代码映射 `_get_title_for_type` + README）：`summary`、`faq`、`study_guide`、`outline`、`podcast`、`timeline`、`glossary`、`quiz`、`infograph`、`ppt`、`mindmap`、`insight`
  - `prompt`：string | null（选填；自定义提示词）
  - `source_ids`：string[]（选填；不传/为空会使用该笔记本下全部 sources）
  - `length`：string（选填；默认 `medium`）
  - `format`：string（选填；默认 `markdown`）
- 出参（成功，`data`）：
  - NoteInfo（由 `get_note_info(created_note)` 返回）

Transform 对 `metadata_` 的影响（以 `NoteInfo.metadata_` 为准）：
- 基础元数据：`{ "length": <length>, "format": <format> }`
- `type=infograph`：
  - 可能包含：`image_url`（生成图片的 Web 路径，形如 `/uploads/<file>`）
  - 或包含：`image_error`
- `type=ppt`：
  - 可能包含：`slides`（字符串数组，每项为 `/uploads/<file>`）
  - 或包含：`image_error`
- `type=insight`：
  - 笔记内容 `content` 为模型生成结果
  - 额外会创建一个名为“洞察报告”的 `Source` 并入库向量（该额外 Source 的 metadata 不会反映到笔记的 `metadata_` 中；笔记 `metadata_` 仍是基础 `{length,format}`）

前端实际请求示例（字段名对照）：
```json
{
  "type": "summary",
  "prompt": null,
  "source_ids": ["<source_id_1>", "<source_id_2>"],
  "length": "medium",
  "format": "markdown"
}
```

### 2.8 聊天（Chat）

#### GET `/api/notebooks/{notebook_id}/chat/sessions`

- Path 参数：
  - `notebook_id`：string
- 出参（成功，`data`，数组）：
  - `data[]`：ChatSessionInfo

ChatSessionInfo 字段：
| 字段 | 类型 | 描述 |
|---|---|---|
| `id` | string | 会话 id |
| `notebook_id` | string | 笔记本 id |
| `title` | string | 会话标题 |
| `created_at` | string | 创建时间 |
| `updated_at` | string | 更新时间 |
| `metadata` | object | 元数据（字段为 `metadata`，来自 `session.metadata_`） |

#### POST `/api/notebooks/{notebook_id}/chat/sessions`

- Path 参数：
  - `notebook_id`：string
- 入参：
  - `title`：string（查询参数，不在 body）
- 出参（成功，`data`）：
  - ChatSessionInfo

#### DELETE `/api/notebooks/{notebook_id}/chat/sessions/{session_id}`

- Path 参数：
  - `notebook_id`：string
  - `session_id`：string
- 出参（成功）：
  - `data`：null

#### POST `/api/notebooks/{notebook_id}/chat/sessions/{session_id}/messages`

- Path 参数：
  - `notebook_id`：string
  - `session_id`：string
- 入参（JSON body，`ChatRequest`）：
  - `message`：string（用户输入）
  - `session_id`：string | null（可选，但后端实际使用 Path 中的 `session_id`）
- 出参（成功，`data`）：
  - `session_id`：string
  - `message`：string（助手回复）
  - `sources`：string[]（命中的 source id 列表）

#### POST `/api/notebooks/{notebook_id}/chat`

这是前端当前主要使用的聊天接口（“发消息并自动建会话/续会话”）。

- Path 参数：
  - `notebook_id`：string
- 入参（JSON body，`ChatRequest`）：
  - `message`：string（必填）
  - `session_id`：string | null（选填）
    - 不传/为空：后端会创建新会话并生成 `session_id`
    - 传入：沿用旧会话（并使用历史 messages）
- 出参（成功，`data`）：
  - `session_id`：string（本次会话 id）
  - `message`：string（助手回复）
  - `sources`：string[]（命中的 source id 列表）

### 2.9 文件信息（文件列表）

#### GET `/api/files/{filename}`

- 鉴权：需要
- Path 参数：
  - `filename`：string（文件名，如 `xxx.pdf`）
- 入参：
  - 无其他参数
- 出参（成功，`data`）：
  - `Content-Type`：string（根据文件后缀映射，如 `image/png` 等；未知为 `application/octet-stream`）
  - `Cache-Control`：string
    - public 笔记：`public, max-age=3600`
    - 私有笔记：`no-cache`
  - `file`：string（后端返回的文件绝对路径；前端通常用于展示或后续处理）

可能错误：
- 404：File not found
- 403：Access denied

> 注：该接口返回的是文件信息 JSON，而不是直接返回文件二进制流。

### 2.10 管理员接口（Admin）

> 需要鉴权，并且当前用户必须满足：`current_user.email == "admin@zuel.edu.cn"`

#### GET `/api/admin/users`

- 入参：
  - 无（由 token 决定用户）
- 出参（成功，`data`，数组）：
  - `data[]` 用户信息：
    - `id`：number
    - `username`：string
    - `email`：string
    - `created_at`：string

#### PUT `/api/admin/users/{user_id}`

- Path 参数：
  - `user_id`：number
- 入参（JSON body，`UpdateRequest`）：
  - `email`：string（必填）
  - `username`：string | null（选填；允许为 null）
  - `password`：string | null（选填；允许为 null）
- 出参（成功，`data`）：
  - 更新后的用户信息：
    - `id`、`username`、`email`、`created_at`

可能错误：
- 404：用户不存在

#### DELETE `/api/admin/users/{user_id}`

- Path 参数：
  - `user_id`：number
- 出参（成功）：
  - `data`：null

可能错误：
- 403：不能删除自己（禁止删除当前 admin 用户）
- 404：用户不存在

#### GET `/public/notebooks/{token}/sources`

#### GET `/public/notebooks/{token}/notes`

- 出参（成功，`data`，数组）：
  - NoteInfo（字段同 `GET /api/notebooks/{notebook_id}/notes`）

## 3. 备注：前端如何调用鉴权/返回值

- 前端在封装 `api(endpoint, options)` 中会把 `Authorization` 头自动加上：
  - `Authorization: Bearer ${token}`
- 前端解析响应时返回值为 `result.data`：
  - 成功时直接使用 `data` 字段（不直接消费 `code/message`）
